# Deployment Design: Spotter

**Date:** 2026-06-15
**Status:** Approved

---

## Overview

Deploy the Spotter ELD Trip Planner as a live full-stack app:
- **Frontend** (React/Vite) → Vercel
- **Backend** (Django/DRF) → Render (free tier, no spin-down issues accepted)

---

## Architecture

```
Browser → Vercel (React/Vite SPA)
               ↓ VITE_API_BASE_URL
          Render (Django + Gunicorn)
               ↓ ORS_API_KEY
          OpenRouteService API
```

---

## Backend — Render

### Why Render

Free tier, GitHub-native deploy, no credit card required. Spins down after 15 min of inactivity (~30s cold start on first request). Acceptable for a portfolio/assessment project.

### Changes Required

**1. Add gunicorn to `spotter-backend/pyproject.toml`**

Add `gunicorn` to the `[project] dependencies` list. Render requires a production WSGI server — Django's `runserver` is not suitable.

**2. Create `spotter-backend/src/app/wsgi.py`**

Standard Django WSGI entry point. Does not currently exist in the project.

```python
import os
from django.core.wsgi import get_wsgi_application

os.environ.setdefault("DJANGO_SETTINGS_MODULE", "app.settings")
application = get_wsgi_application()
```

**3. Update `spotter-backend/src/app/settings.py`**

Change the `DEBUG` default from `"true"` to `"false"` so production is safe if the env var is missing.

```python
DEBUG = os.getenv("DEBUG", "false").lower() == "true"
```

**4. Create `spotter-backend/render.yaml`**

Declares the Render web service configuration, including build/start commands and environment variable stubs.

```yaml
services:
  - type: web
    name: spotter-backend
    runtime: python
    pythonVersion: "3.11"
    buildCommand: pip install uv && uv sync
    startCommand: uv run gunicorn app.wsgi:application --bind 0.0.0.0:$PORT --workers 2
    envVars:
      - key: DEBUG
        value: false
      - key: SECRET_KEY
        generateValue: true
      - key: ORS_API_KEY
        sync: false
```

### Environment Variables (set in Render dashboard)

| Key | How to set |
|---|---|
| `ORS_API_KEY` | Manually — paste from `.env` |
| `SECRET_KEY` | Auto-generated by Render |
| `DEBUG` | `false` (declared in render.yaml) |

`ALLOWED_HOSTS` stays `["*"]` — acceptable for a demo deployment.
`CORS_ALLOW_ALL_ORIGINS = True` — stays as-is; restricting to Vercel domain is a post-launch concern.

### Deploy Order

1. Connect GitHub repo to Render
2. Point Render at `spotter-backend/` as root directory (or let render.yaml handle it)
3. Add `ORS_API_KEY` manually in the Render dashboard
4. Deploy → note the service URL (e.g. `https://spotter-backend.onrender.com`)

## Frontend — Vercel

### Changes Required

**1. Update `spotter-frontend/vercel.json`**

Add a SPA catch-all rewrite so React Router routes work on direct navigation / refresh.

```json
{
  "buildCommand": "pnpm build",
  "outputDirectory": "dist",
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

**2. Set `VITE_API_BASE_URL` in Vercel dashboard**

Points the frontend at the live Render backend. Set this before deploying the frontend.

| Key | Value |
|---|---|
| `VITE_API_BASE_URL` | `https://spotter-backend.onrender.com` (Render URL from step above) |

### Deploy Order

1. Get the Render backend URL first
2. Add `VITE_API_BASE_URL` in Vercel project settings → Environment Variables
3. Connect GitHub repo to Vercel (or push to trigger auto-deploy)
4. Vercel auto-detects `vercel.json` and builds/deploys

---

## Deploy Sequence (end to end)

1. Make all code changes (gunicorn dep, wsgi.py, settings.py, render.yaml, vercel.json)
2. Commit and push to GitHub
3. Create Render web service → root dir: `spotter-backend` → add `ORS_API_KEY` → deploy
4. Copy Render URL
5. Create Vercel project → root dir: `spotter-frontend` → set `VITE_API_BASE_URL` → deploy
6. Smoke test: open Vercel URL, plan a trip, verify log sheets render

---

## Out of Scope

- Custom domains
- HTTPS (both platforms provide it automatically)
- Restricting CORS/ALLOWED_HOSTS to specific domains
- Database persistence (app is stateless by design)
- CI/CD pipelines beyond GitHub auto-deploy
